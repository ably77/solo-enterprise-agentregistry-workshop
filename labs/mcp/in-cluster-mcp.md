# In-Cluster MCP Server (Bring Your Own)

The [Solo Docs](solo-docs-mcp.md) and [DeepWiki](deepwiki-mcp.md) labs register MCP servers that
already run somewhere on the public internet. This lab covers the other common case: **you run the
MCP server yourself** as an ordinary Kubernetes `Deployment` + `Service`, then register its
**in-cluster Service URL** in the Agentregistry catalog and expose it through Agentgateway.

We use a real, public MCP image — an **arXiv** academic-paper search server — but the pattern is
identical for any MCP server image you build and host yourself.

The only thing that changes from the remote labs is the catalog entry's `remote.url`: instead of
`https://search.solo.io/mcp`, it points at `http://mcp-airxiv.mcp.svc.cluster.local:80/mcp`.
Everything downstream — Virtual runtime, generated child route + backend, gateway endpoint — is the
same.

## Lab Objectives

- Deploy a self-hosted MCP server (`Deployment` + `Service`) in a `mcp` namespace
- Catalog it in Agentregistry by its in-cluster Service URL
- Deploy it to the `virtual-default` runtime and expose it through Agentgateway
- Call its tools at `/registry/arxiv`

## Pre-requisites

- [001 - Installation](../../001-installation.md) complete
- Optional: to run this lab's calls from a browser instead of `curl`, use the
  [MCP Client UI](mcp-client-ui.md) and pick the **arXiv (in-cluster)** endpoint.
- Shell context:

```bash
export PATH=$HOME/.arctl/bin:$PATH
source ~/.are-keycloak-env
export AR_IP=$(kubectl get svc agentregistry-enterprise-server -n agentregistry-system \
  -o jsonpath='{.status.loadBalancer.ingress[0].ip}{.status.loadBalancer.ingress[0].hostname}')
export ARCTL_API_BASE_URL="http://${AR_IP}:12121"
```

> **Third-party image notice:** `ably7/airxiv-mcp` is a community image, used here for convenience.
> For anything beyond a workshop, build/pin your own MCP server image and scan it.

## Architecture

```
client → http://<gateway-LB>/registry/arxiv          (Agentgateway, port 80)
           → child HTTPRoute + EnterpriseAgentgatewayBackend   (generated by agentregistry)
              → http://mcp-airxiv.mcp.svc.cluster.local:80/mcp  (YOUR in-cluster Deployment)
```

## 1. Deploy the MCP Server In-Cluster

[`assets/mcp/in-cluster/arxiv-deployment.yaml`](../../assets/mcp/in-cluster/arxiv-deployment.yaml)
is a plain `Deployment` + `Service` (no agentregistry or gateway CRDs) serving streamable-HTTP on
port 8000 at `/mcp`:

```bash
cat assets/mcp/in-cluster/arxiv-deployment.yaml
kubectl create namespace mcp
kubectl apply -f assets/mcp/in-cluster/arxiv-deployment.yaml
kubectl rollout status deployment/mcp-airxiv -n mcp
```

(Optional) sanity-check the server directly from inside the cluster:

```bash
kubectl port-forward -n mcp svc/mcp-airxiv 8800:80 &
sleep 2
curl -s -X POST -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-06-18","capabilities":{},"clientInfo":{"name":"curl","version":"0.0.1"}}}' \
  http://localhost:8800/mcp | head -2
kill %1 2>/dev/null
```

You should see a `200`/`text/event-stream` initialize response. Now we'll front it with Agentgateway
instead of port-forwarding.

## 2. Parent Gateway and Route

Same parent Gateway the other MCP labs use. Skip if you already applied it.

```bash
cat assets/mcp/agentgateway/parent-gateway-and-route.yaml
kubectl apply -f assets/mcp/agentgateway/parent-gateway-and-route.yaml
kubectl -n agentgateway-system get gateway agentregistry-gateway -w
# Wait for PROGRAMMED=True and an ADDRESS, then Ctrl-C
```

## 3. Catalog the In-Cluster Server

The catalog entry points at the in-cluster Service DNS name:

```bash
cat assets/mcp/in-cluster/arxiv-mcp.yaml
arctl apply -f assets/mcp/in-cluster/arxiv-mcp.yaml
arctl get mcp arxiv-incluster-mcp --tag latest -o yaml | grep -A3 remote:
```

```yaml
  remote:
    type: streamable-http
    url: http://mcp-airxiv.mcp.svc.cluster.local:80/mcp
```

## 4. Deploy to the Virtual Runtime

```bash
cat assets/mcp/in-cluster/arxiv-mcp-deploy.yaml
arctl apply -f assets/mcp/in-cluster/arxiv-mcp-deploy.yaml
sleep 5
arctl get deployment arxiv-incluster-agw -o yaml | grep -E "reason:|url:"
```

Expect `reason: DeployedViaAgentgateway` and `url: http://<gateway-address>/registry/arxiv`.

> **Ordering gotcha:** if the parent Gateway wasn't `PROGRAMMED` when you applied this, the
> Deployment sticks at `NoAcceptedListener` and won't self-heal. Recover with
> `arctl delete deployment arxiv-incluster-agw` then re-apply (see [Solo Docs lab](solo-docs-mcp.md#4-deploy-the-mcp-to-the-virtual-runtime)).

## 5. Call It Through the Gateway

```bash
export AGW_ADDRESS=$(kubectl -n agentgateway-system get gateway agentregistry-gateway \
  -o jsonpath='{.status.addresses[0].value}')

# initialize → capture session
export SID=$(curl -s -D - -o /dev/null -X POST \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-06-18","capabilities":{},"clientInfo":{"name":"curl","version":"0.0.1"}}}' \
  "http://${AGW_ADDRESS}/registry/arxiv" | awk -F': ' 'tolower($1)=="mcp-session-id"{print $2}' | tr -d '\r')

H=(-H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
   -H "mcp-session-id: ${SID}" -H "MCP-Protocol-Version: 2025-06-18")
curl -s -o /dev/null -X POST "${H[@]}" -d '{"jsonrpc":"2.0","method":"notifications/initialized"}' "http://${AGW_ADDRESS}/registry/arxiv"

# list tools
curl -s -X POST "${H[@]}" -d '{"jsonrpc":"2.0","id":2,"method":"tools/list"}' \
  "http://${AGW_ADDRESS}/registry/arxiv" | sed 's/^data: //' | jq -r '.result.tools[].name'
```

Expected:

```
search_arxiv
get_paper
get_recent
search_papers
fetch_pdf
```

Call the `search_arxiv` tool:

```bash
curl -s -X POST "${H[@]}" \
  -d '{"jsonrpc":"2.0","id":3,"method":"tools/call","params":{"name":"search_arxiv","arguments":{"keyword":"retrieval augmented generation","max_results":3}}}' \
  "http://${AGW_ADDRESS}/registry/arxiv" | sed 's/^data: //' | jq -r '.result.content[0].text' | head -20
```

You get arXiv results, served from **your** in-cluster pod, through the gateway.

## Cleanup

```bash
arctl delete deployment arxiv-incluster-agw
arctl delete mcp arxiv-incluster-mcp --tag latest
kubectl delete -f assets/mcp/in-cluster/arxiv-deployment.yaml
kubectl delete namespace mcp --ignore-not-found
# Only if you're done with the parent Gateway (other MCP labs share it):
kubectl -n agentgateway-system delete httproute remote-mcp-delegate --ignore-not-found
kubectl -n agentgateway-system delete gateway   agentregistry-gateway   --ignore-not-found
```
